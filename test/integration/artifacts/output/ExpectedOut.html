PARAMETERLESS TESTS

Parameterless tests are the simplest supported form of tests.

Passing parameterless tests are written in a simple form: T1.
Failing tests will always T2 T2_ACTUAL show the expected and real results.

SINGLE PARAMETER TESTS

Single parameter tests support text input declaration like links.
Assigning a variable is done like so: V1
This will assign the value V1 to the parameter named var1 in the background map.
Reassigning a variable is done by reusing the statement: V2. Parameter var1 now has a value of V2.
This variable can now be passed to a test, either passing or failing.
T3
T4 T4_ACTUAL
